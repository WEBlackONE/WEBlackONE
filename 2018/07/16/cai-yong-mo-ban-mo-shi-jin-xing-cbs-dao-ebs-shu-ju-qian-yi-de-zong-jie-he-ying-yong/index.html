一.模板模式进行cbs.deal到ebs数据迁移的总结和应用

1.预热部分

1.1.简单粗暴理解一下模板模式

有一些操作，他们的流程都是固定的，就那么几步，我们把固定的流程提取出来，做成一个模板，之后的具体到某个操作，直需要在流程中丰富这个操作的细节即可。

1.2.举个例子：

比方我们平时炒菜就那么几步：1.起锅烧油—>>>2.放菜3.炒—>>>4.出锅装盘

把这个流程提起成一个模板，

如果我们要抄青菜，在步骤2中就放入青菜，出锅的就是炒青菜

如果要炒土豆丝，就在步骤2中放入土豆丝，出锅的就是炒土豆丝

1.3.一个不知名的小demo

1.3.1.确认流程
public abstract class 炒菜模板_baseHandler  {
    //定义流程步骤
    public void 炒菜流程_handle(){
           // 1
            起锅烧油();
            //2
            放菜();
            //3
            炒();
            //4
            出锅装盘();
    }
    public void 起锅烧油(){
        System.out.println("起锅烧油操作-->>");
    }
    //需要子类重写具体实现
    public abstract void 放菜();

    public void 炒(){
        System.out.println("炒菜时间到-->>");
    }
    public void 出锅装盘(){
        System.out.println("出锅装盘!");
    }
}
1.3.2.一个具体的实现
public class 炒土豆丝_土豆丝Handler extends 炒菜模板_baseHandler{
    @Override
    public void 放菜() {
        System.out.println("放入土豆丝");
    }
}

总结：

  • 什么情况下可以应用模板模式：

    • 1.有固定的流程

  • 模板模式的实践流程：

    • 1.要做个什么事儿？（要炒菜）

    • 2.要怎么做？（炒菜的流程）

    • 3.具体要做个什么事情？（要炒土豆丝）

2.重头戏

下面就按1的总结流程3步骤来实现我们这次cbs->ebs数据迁移的工作。

2.1.要做个什么事儿？

把cbs.deal库中写好的视图提取出的数据导入到业财的ebs库中。

看起来有点太简单粗暴，就好像占山为王的老土匪（ebs）直接把村长的女儿抢到手（cbs.deal.视图），有点无耻。

更妥当的做法应该是娶的时候和老村长提个亲（把主表的状态更新为syncing），告诉村长我要娶了。最好是合法一下，到民政局去登记（记录在批量表），这么重要的日子也应该纪念一下（记录在日志表），最后事情的结果怎么样，也应该告知一声老村长，顺风顺水，完美成功（就把主表的状态更新为synced），如果是婚检检出了遗传病，这事儿就凉了（把主表的状态更新为error），下面我们对迁移步骤做一次升级处理：

升级处理以后我们的流程也基本确定下来了。

2.要怎么做？

2.1.迁移数据的流程：
  1. 获取相应的视图数据
  2. ebs批表插入数据
  3. 更新原始主表的同步状态（syncing）
  4. 传输数据到ebs数据表中
    1. 传输成功
      1. 更新主表的同步状态（synced）
    2. 传输失败
      1. 更新主表的同步状态（error）
  5. 记录日志

3.具体要做个什么事情？

到某一件具体的事情就是实例化的过程。实现的过程就是把流程中不能确定的信息（抽象概括的信息）确认下来的过程

举例：要迁移合同信息的数据

流程中不能确定的信息有那些：

1.获取相应的视图数据

​ 1.我们确认下来获取合同的视图是 ebs_ar_contract

2.需要更新的主表不确定

​ 1.我们要确认下来合同视图的主表是哪一个ebs_ar_contract视图对应的主表是 deal.cr_case_info

序号 视图名 视图注解 库表名 表注释 备注
01 ebs_ar_contract 合同信息 deal.cr_case_info 案件信息
02 ebs_ar_receipts_receipt 收款信息 deal.fin_receipt 实收明细
03 ebs_ar_receipts_statement 收款信息 deal.fin_receipt_statement 实收流水
04 ebs_ar_refund_espense 退款信息 deal.fin_refund 退款 待确认
05 ebs_ar_refund_statement 退款信息 deal.fin_refund_statement 退款
06 ebs_ar_revenue_invoice 收入信息 deal.fin_recv_invoice_info 开票信息
07 ebs_ar_revenue_receive 收入信息 deal.cr_receivable 应收明细
08 ebs_gl_payment_expense 业务付款 deal.fin_payment 付款
09 ebs_gl_payment_payment 业务付款 deal.fin_payment_statement 付款明细

3.迁移到业财ebs的表中不确定

​ 1.确定业财的合同表 CUX_EBS_AR_CONTRACT_IFACE

到代码中就是确定：

public abstract class BaseHandler<D extends BaseDealEntity, E extends BaseEbsEntity,R extends BaseCbsEntity>

1.BaseDealEntity 获取视图实体 eg：子类> 合同：EbsArContractEntity

2.BaseEbsEntity 业财ebs表的实体 eg: 子类>合同：CuxEbsArContractIfaceEntity

3.BaseCbsEntity 需要更新的主表实体 子类>合同：CrCaseInfo

需要留意的几个地方：

1.视图必须带有主表的id（sync_id）,用于之后的主表状态更新

2.多数据源，不是service实现类中，最好不要写lambda表达式，可能会识别不出来具体到哪一个库。
